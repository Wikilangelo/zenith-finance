# Coding Guidelines

## Principles

- readability first
- avoid overengineering
- strong typing
- simple abstractions
- reusable components

## Documentation

Every complex function must explain:
- what it does
- why it exists
- important edge cases

## Naming

Use descriptive names.

Avoid abbreviations unless universally understood.

## Components

Components must:
- have one responsibility
- remain reusable
- avoid hidden side effects

## AI Generated Code

AI-generated code must:
- pass lint
- pass tests
- be reviewed
- remain understandable by junior developers
